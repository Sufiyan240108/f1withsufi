# Contributing to f1withsufi

First off, thank you for considering contributing to **f1withsufi**! It's people like you that make the F1 data community great.

## How Can I Contribute?

### Reporting Bugs
- Use the GitHub Issue Tracker.
- Describe the bug clearly and provide steps to reproduce.
- Mention your environment (OS, Python/Node versions).

### Suggesting Enhancements
- Open a "Feature Request" in the GitHub Issues.
- Explain why this feature would be useful for F1 analytics.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. Install dependencies for both `frontend` and `backend`.
3. If you've added code that should be tested, add tests.
4. Ensure the test suite passes.
5. Make sure your code follows the existing style (Linting).
6. Issue the PR!

## Local Setup

Refer to the [README.md](README.md) for detailed local setup instructions.

### Coding Standards
- **Backend (Python):** PEP 8 compliant. Use type hints where possible.
- **Frontend (React):** Functional components with Hooks. Keep CSS modular.
